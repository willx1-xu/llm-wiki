# 内容矩阵策略

## 核心理念

不是写孤立的文章，而是**系统性地覆盖一个领域的全部搜索意图**。

## Topic Cluster 模型

```
        [Pillar Page: 独立站 SEO 完整指南]
                   |
    ┌──────────────┼──────────────┐
[关键词研究]  [GitHub SEO]  [内容策略]  [外链建设]
    |              |              |
 [长尾FAQ]    [README优化]   [文章模板]  [平台分发]
```

- Pillar Page：3000+ 字，全面覆盖
- Cluster Page：500-1500 字，聚焦单一问题
- 全部互相链接

## 内容类型

| 类型 | 目标 | 示例 |
|------|------|------|
| FAQ | 捕获长尾问题 | "独立站需要备案吗" |
| 教程 | 实操指导 | "GitHub 项目 SEO 优化步骤" |
| 评测/对比 | 商业意图 | "Vercel vs Netlify 独立站部署对比" |
| 案例 | 信任建设 | "我的独立站 3 个月从 0 到 1000 UV" |

## 发布节奏
- 初期集中出 FAQ（量优先，覆盖长尾）
- 中期打磨 Pillar Page（质优先，竞争核心词）
- 长期持续更新（Google 偏爱活跃站点）
